# [내일배움캠프 Spring_3기] CH2 커머스 과제

상품/카테고리/고객/주문/재고라는 커머스의 핵심 도메인을 객체로 정의하고, 객체들이 협력하는 구조로 기능을 확장했습니다.  

- 개발 환경: Java 17, IntelliJ IDEA, Gradle
- 패키지 기준:
  - 필수 Step: `kr.spartaclub.com.example.commerce.step*`
  - 도전 과제: `kr.spartaclub.com.example.commerce.challenge`

---

## 프로젝트 목표

- Java 문법(컬렉션, 예외 처리, 캡슐화)을 실제 프로그램 구조에 적용
- 객체지향 관점에서 역할과 책임(R&R)을 분리하여 확장 가능한 구조 설계
- Enum, 람다, 스트림을 활용한 기능 구현 경험

---

## 주요 기능

### Step 1. 상품 목록 출력
- `Product` 클래스로 상품 데이터 구조화
- `List<Product>`로 상품 목록 구성 및 콘솔 출력

### Step 2. 흐름 제어 분리
- `CommerceSystem` 클래스를 도입하여 입력/출력/반복 로직을 분리
- `main`에서는 시스템 생성 및 시작만 담당

### Step 3. 카테고리/고객 도입
- `Category`로 카테고리별 상품 관리
- `Customer`로 고객 정보(이메일/등급/누적 주문 금액) 구조화

### Step 4. 캡슐화 적용
- 주요 도메인 객체의 필드 직접 접근 제한
- Getter/Setter(필요한 경우) 기반으로 상태 변경 제어

---

## 도전 기능

### Lv1. 장바구니 및 주문 기능
- `Cart / CartItem` 도입으로 장바구니 관리
- 상품 선택 후 수량 입력 및 유효성 검사
- 주문 확정 시 재고 차감, 장바구니 초기화
- 주문 취소 시 장바구니 초기화
- 장바구니 비어있을 때 주문 메뉴(장바구니 확인/주문 취소) 접근 제한

### Lv2. 관리자 모드
- 관리자 비밀번호 인증(3회 실패 시 메인 메뉴 복귀)
- 상품 추가 / 수정 / 삭제
- 카테고리 선택 후 상품 관리
- 상품 삭제 시 장바구니에 담긴 동일 상품도 함께 제거

### Lv3. Enum, 람다 & 스트림 활용
- `CustomerGrade` Enum으로 등급별 할인율 관리 (BRONZE/SILVER/GOLD/PLATINUM)
- 주문 시 고객 등급에 따른 할인 적용 및 결제 금액 계산
- 스트림 기반 가격 필터링(100만원 이하 / 초과)
- 스트림 기반 장바구니 특정 상품 제거
